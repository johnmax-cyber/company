## Implementation Steps

### Step 0a: Implement Supabase Fallback Enhancement

**File:** `src/services/product-service.js`

Enhance all fetch methods with explicit try/catch and console.warn messages:

```javascript
async getProducts() {
  const cached = store.get(CACHE_KEY);
  if (cached && (Date.now() - cached.timestamp < CACHE_TTL)) {
    return cached.data;
  }

  try {
    const { data, error } = await supabase
      .from('products')
      .select('*')
      .eq('in_stock', true);

    if (error) throw error;
    if (!data || data.length === 0) throw new Error('No products returned from Supabase');

    store.set(CACHE_KEY, { data, timestamp: Date.now() });
    return data;
  } catch (error) {
    console.warn('Supabase unavailable, loading from products.json fallback:', error.message);
    
    try {
      const response = await fetch('/products.json');
      if (!response.ok) throw new Error(`HTTP ${response.status}: ${response.statusText}`);
      const { products } = await response.json();
      if (!products || !Array.isArray(products)) throw new Error('Invalid products.json format');
      
      const filtered = products.filter(p => p.inStock !== false) || [];
      store.set(CACHE_KEY, { data: filtered, timestamp: Date.now() });
      console.info(`Loaded ${filtered.length} products from products.json fallback`);
      return filtered;
    } catch (jsonError) {
      console.error('Both Supabase and products.json failed:', jsonError);
      if (cached) {
        console.warn('Using stale cache as last resort');
        return cached.data;
      }
      return [];
    }
  }
}
```

Apply same pattern to: `getAllProducts()`, `getProductById()`, `searchProducts()`, `getProductsByCategory()`.

**Validation:** Turn off network, reload `/shop` → products still render from JSON fallback.

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Image URLs break (Unsplash changes) | UI shows broken images | Use Unsplash Source with `?w=800&fit=crop` which redirects to stable CDN |
| Duplicate product IDs | Cart/API errors | Script validation ensures uniqueness |
| JSON syntax error | App fails to load | Run `node -e "require('./products.json')"` to validate |
| Too many products slows initial load | Performance degradation | Keep total < 100, images lazy-loaded, cached in localStorage |
| Prices inconsistent with real market | Unrealistic test | Research actual Kenyan price points for clothing/books |
