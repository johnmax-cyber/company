## Detailed Changes

### Fix 1: Remove ADMIN_PASSWORD from api/config.js
**File:** `api/config.js`
- **Current behavior:** Exposes `window.__ADMIN_PASSWORD__` in addition to SUPABASE_URL and SUPABASE_ANON_KEY
- **Desired state:** Only sets `window.__SUPABASE_URL__` and `window.__SUPABASE_ANON_KEY__`
- **Action:** 
  - Remove `adminPassword` variable (line 5)
  - Remove `window.__ADMIN_PASSWORD__` from response string (line 7)
  - Keep only SUPABASE_URL and SUPABASE_ANON_KEY

**File:** `admin.html`
- **Current behavior:** No login form, directly shows admin panel, no auth check
- **Desired state:** Email/password login form before dashboard, authenticated via Supabase
- **Actions:**
  1. Add login form with email + password fields before the admin dashboard content
  2. On form submit, call `supabase.auth.signInWithPassword({ email, password })`
  3. On success:
     - Store returned JWT token from `data.session.access_token`
     - Use as `Authorization: Bearer <token>` on all fetch calls to `/api/admin`
     - Hide login form, show dashboard
  4. On failure: Show "Invalid credentials" error message
  5. Add logout button that calls `supabase.auth.signOut()` and hides dashboard, shows login form
  6. Wrap admin panel in a container that is hidden by default, shown after login
  7. Check for existing session on page load (optional - can auto-login if session exists)

### Fix 3: Create missing src/pages modules

**File:** `src/pages/shop.js` (NEW)
- Class: `ShopPage`
- Imports: `productService` from '../services/product-service.js', `cartService` from '../services/cart-service.js'
- Methods:
  - `async init()`:
    - Show loading spinner in #productsGrid (if exists)
    - Fetch all products using `productService.getProducts()`
    - OR check URL params for `?category=X&subcategory=Y` and use `productService.getProductsByCategory()` or `productService.searchProducts()`
    - Parse URL search params from `window.location.search`
    - Render products into #productsGrid
    - Add "Add to Cart" buttons that call `cartService.addItem()`
    - Wire #searchInput to filter products if it exists
    - Show error message on failure
    - Show loading state while fetching

**File:** `src/pages/cart.js` (NEW)
- Class: `CartPage`
- Imports: `cartService` from '../services/cart-service.js', `orderService` from '../services/order-service.js'
- Methods:
  - `async init()`:
    - Render cart items from `cartService.getItems()` into #cartItems
    - Calculate and show subtotal and total
    - Wire quantity update inputs to update quantities
    - Wire remove buttons to remove items
    - Wire checkout form (name, phone, payment method) to `orderService.createOrder()`
    - On successful order: clear cart with `cartService.clearCart()`, show confirmation message

**File:** `src/pages/product.js` (NEW)
- Class: `ProductPage`
- Imports: `productService` from '../services/product-service.js', `cartService` from '../services/cart-service.js'
- Methods:
  - `async init()`:
    - Get product id from URL param `?id=X` using `URLSearchParams`
    - Fetch product using `productService.getProductById(id)`
    - Render name, price, description, image, icon into the page
    - Wire "Add to Cart" button to `cartService.addItem()`
    - Show error message if product not found (404)

### Fix 4: Update src/main.js for page routing
**File:** `src/main.js`
- **Current behavior:** Always initializes HomePage
- **Desired state:** Detect page path and initialize appropriate class
- **Actions:**
  - Get current path from `window.location.pathname`
  - If `/shop` in path → `new ShopPage().init()`
  - If `/cart` in path → `new CartPage().init()`
  - If `/product` in path → `new ProductPage().init()`
  - Else → `new HomePage().init()` (existing)
  - Remove the duplicate DOMContentLoaded listener from home.js (it has one too - but that file shouldn't be modified per user request, only main.js)
