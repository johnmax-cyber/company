# Faith & Fashion Nairobi - Upgrade Plan

## Project Analysis Summary

### Security Vulnerabilities
1. **Exposed Secrets**: SUPABASE_URL, SUPABASE_ANON_KEY, and ADMIN_PASSWORD are exposed in frontend code via config.js
2. **Unprotected API Routes**: /api/admin.js lacks authentication checks - accessible to anyone
3. **Missing Rate Limiting**: No protection against brute force or abuse on admin endpoints
4. **Overly Permissive CORS**: Access-Control-Allow-Origin: * in api/admin.js
5. **Sensitive Data Exposure**: Error messages may leak internal details

### Performance Issues
1. **Monolithic JavaScript**: main.js (773 lines) mixes UI logic, data fetching, and DOM manipulation
2. **Duplicate Logic**: Cart handling duplicated between index.html and main.js
3. **Inefficient Filtering**: Products filtered/sorted multiple times unnecessarily
4. **No Image Optimization**: Beyond basic lazy loading in hero section
5. **No Caching**: Repeated API calls for same data

### Bad Coding Patterns
1. **Tight Coupling**: Business logic mixed with DOM manipulation
2. **Global Namespace Pollution**: Multiple variables attached to window object
3. **Inconsistent Error Handling**: Mix of try/catch, error checking, and alerts
4. **Mixed Concerns**: Single files handle multiple responsibilities

### Redundant Logic
1. **Cart Functions**: addToCart, updateCartCount, saveCart duplicated
2. **Product Loading**: Similar logic in index.html and main.js
3. **Utility Functions**: sanitizeInput, escapeHtml defined in multiple places

### PART 2: Frontend Improvements
1. **Modular JavaScript**: ES6 modules with proper imports/exports
2. **Lazy Loading**: Implement for all product images
3. **State Management**: Simple pub/sub pattern for cart updates
4. **Loading/Error States**: Consistent UI feedback
5. **DOM Optimization**: Event delegation, minimal reflows
6. **CSS Optimization**: Remove unused styles, improve specificity

### PART 3: Security Hardening
1. **Environment Variables**: Move all secrets to Vercel env vars
2. **API Protection**: 
   - Require JWT tokens for admin endpoints
   - Validate tokens with Supabase
   - Implement role-based access (admin vs user)
3. **Input Validation**: Strict validation on all API inputs
4. **Rate Limiting**: Basic implementation for admin endpoints
5. **Error Handling**: Generic error messages in production
6. **Frontend Secrets**: Remove all secrets from client-side code

### PART 4: Supabase Optimization
1. **Schema Improvements**:
   - Add indexes on frequently queried columns (category, subcategory, price)
   - Add full-text search capability for product names
   - Improve JSONB indexing for order items
2. **Row Level Security**:
   - Refine policies for better granularity
   - Implement user-specific access where needed
   - Ensure service role is only used server-side
3. **Query Optimization**:
   - Use select columns explicitly instead of *
   - Add pagination to product listings
   - Implement proper filtering at database level

### PART 5: API Improvements
1. **Authentication Middleware**: Verify Supabase JWT tokens
2. **Role-Based Access**: Separate admin/user endpoints
3. **Consistent Responses**: Standard JSON format with status codes
4. **Validation Layer**: Centralized input validation
5. **Logging**: Basic request logging for debugging

### PART 6: Performance Optimization
1. **API Call Reduction**: 
   - Cache product data in localStorage with TTL
   - Implement request deduplication
2. **Prefetching**: Anticipate user navigation
3. **Bundle Optimization**: Code splitting for routes
4. **Image Optimization**: Proper sizing, modern formats
5. **Critical Rendering Path**: Prioritize above-the-fold content

### PART 7: Vercel Configuration
1. **Function Optimization**:
   - Set appropriate memory limits (256MB-512MB)
   - Optimize duration (10s for most functions)
   - Enable caching where appropriate
2. **Routing**: Clean API routes, proper redirects
3. **Environment Variables**: Secure setup guide
4. **Monitoring**: Enable logging and alerts

## Implementation Order
1. Create new folder structure
2. Move and refactor existing code into modules
3. Implement security hardening (env vars, API protection)
4. Optimize Supabase schema and queries
5. Improve frontend performance (lazy loading, caching)
6. Update Vercel configuration
7. Final testing and validation

## Environment Variables Required
- SUPABASE_URL
- SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY (server-only)
- ADMIN_SECRET (for additional API protection)
- NODE_ENV (development/production)

## Deployment Instructions
1. Set environment variables in Vercel dashboard
2. Deploy to Vercel (automatically detects vercel.json)
3. Verify Supabase connection and RLS policies
4. Test all endpoints for security and functionality
5. Monitor performance and error rates
